<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| settings service → portfolio service | App-wide configuration influences every returned valuation |
| portfolio service → pricing services | Exact-money transformation occurs here and precision loss would corrupt totals |
| backend DTO mapper → frontend consumers | Incorrect serialization can leak internal shapes or lose precision |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | T | `backend/src/pricing/valuation.service.ts` | mitigate | Use Decimal/string arithmetic only and forbid canonical finance calculations through JS `number`. |
| T-01-08 | I | `backend/src/portfolio/portfolio.mapper.ts` | mitigate | Map explicit DTOs and serialize only approved finance fields needed for D-04 to D-06 instead of returning ORM entities. |
| T-01-09 | T | `backend/src/portfolio/portfolio.service.ts` | mitigate | Read the active base currency from `SettingsService` for every portfolio response so totals cannot diverge from the persisted app-wide setting. |
</threat_model>
